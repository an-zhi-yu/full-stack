# 博客前端代码规范

> 本仓库（`frontend/blog`）内所有前端代码按此规范编写，便于协作与维护。

---

## 一、技术栈

| 类别     | 选型           | 说明           |
|----------|----------------|----------------|
| 框架     | React 18       | 函数组件 + Hooks |
| 路由     | React Router 6 | 声明式路由     |
| 状态     | Zustand        | 全局/跨组件状态 |
| UI       | Ant Design 5   | 组件与主题     |
| 请求     | Axios          | 封装在 `src/api/request.ts` |
| 语言     | TypeScript     | 严格模式       |
| 构建     | Vite           | 开发与打包     |
| 规范     | ESLint + Prettier | 统一风格   |
| 提交流程 | Husky + lint-staged | 提交前自动校验 |
| 测试     | Vitest + React Testing Library | 单元/组件测试 |

---

## 二、目录约定

```
src/
├── api/           # 接口封装（request、各模块 API）
├── components/    # 通用组件（按功能分子目录，如 layout、transition）
├── hooks/         # 自定义 Hooks
├── pages/         # 页面组件（与路由一一对应）
├── routes/        # 路由配置
├── store/         # Zustand 状态
├── styles/        # 全局样式、主题、CSS 变量
└── test/          # 测试 setup、全局 mock
```

- **组件**：能复用的放 `components/`，页面专属的放对应 `pages/xxx/` 下。
- **样式**：优先用 **CSS Modules**（`*.module.css`），全局变量与主题在 `styles/global.css`、`styles/theme.ts`。

---

## 三、命名

| 类型       | 规范               | 示例                |
|------------|--------------------|---------------------|
| 组件文件   | PascalCase         | `HeaderBar.tsx`     |
| 非组件 TS  | camelCase          | `request.ts`        |
| CSS Module | 与组件同名的短横线 | `HeaderBar.module.css` |
| 组件名     | PascalCase         | `<HeaderBar />`     |
| 变量/函数  | camelCase          | `getUserInfo`       |
| 常量       | UPPER_SNAKE 或 camelCase | `API_BASE`   |
| 类型/接口  | PascalCase         | `interface UserInfo` |

---

## 四、组件与 Hooks

- **函数组件**：使用函数声明或 `const Component = () => {}`，导出用 `export default` 或具名导出。
- **Props**：必须写类型（`interface XxxProps`），禁止 `any`。
- **状态**：简单状态用 `useState`，跨组件用 Zustand；避免无意义的全局状态。
- **副作用**：异步、订阅、DOM 用 `useEffect`，注意依赖与清理。

---

## 五、样式与主题

- 主题色、间距等统一用 **CSS 变量**（`styles/global.css` 的 `:root`）和 **Ant Design 的 ConfigProvider theme**（`styles/theme.ts`）。
- 深色/亮色通过顶部「深色」开关切换，逻辑在 `store/theme.ts`。
- 需要动效时：优先用 **CSS 动画**（如 `global.css` 的 `.card-glow`、`.page-enter`），复杂动效再考虑额外库。

---

## 六、请求与接口

- 所有 HTTP 请求走 `src/api/request.ts` 导出的 `request`（基于 axios）。
- 按业务在 `api/` 下拆文件，如 `api/post.ts`，方法返回类型明确写出。

---

## 七、测试

- 单测/组件测放在 **同目录** 或 `__tests__`，文件名：`*.test.tsx` 或 `*.spec.tsx`。
- 运行：`pnpm test`（监听）、`pnpm test:run`（单次）、`pnpm test:coverage`（覆盖率）。
- 新页面或核心组件建议补基础用例（渲染、关键交互）。

---

## 八、提交前

- 执行 `pnpm lint`、`pnpm format:check`，修复报错。
- 提交时 Husky 会跑 **lint-staged**（对暂存文件执行 eslint --fix 与 prettier --write），保持代码风格一致。

---

## 九、扩展与动画

- **自定义主题**：在 `styles/theme.ts` 增 token 或新 theme 对象，在 `store/theme` 中扩展主题键（如亮/暗/其他色系），再在 App 里按主题注入 ConfigProvider。
- **动画**：页面进入用 `PageTransition` 包裹；卡片悬停用 `card-glow` 类；需更复杂动画时可引入 `framer-motion` 等，并在此文档补充约定。

以上规范随项目演进可增删，有疑问以本文件为准。
