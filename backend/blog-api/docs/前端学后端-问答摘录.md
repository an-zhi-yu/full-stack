# 前端学后端：常见问题摘录（与本仓库代码对照）

> 以下为学习过程中反复出现的点，便于和 **`blog-api`** 源码对照，非标准考试答案，以「能联调、能维护」为准。

---

## 1. Controller 里为什么既有 `Map` 又有「先定义类」

- **返回 `Map`**：快速 demo，Jackson 也能序列化成 JSON。  
- **返回专用类 / record / DTO**：字段名稳定、可复用、IDE 可跳转，**生产项目优先**。  
- **健康检查**等若返回固定结构，定义 `HealthPayload` 一类更清晰。

---

## 2. `@PathVariable` 和 `@RequestBody` 前端怎么调

- **PUT `/api/v1/users/{id}`**：URL 里带 id，body 里是 JSON（字段与 `UserUpdateDTO` 对齐）。  
- axios：`axios.put(\`/api/v1/users/${id}\`, payload)`。

---

## 3. 为什么有的项目很少用 PUT/DELETE

- 网关、防火墙、老客户端只放行 GET/POST。  
- 团队习惯「全部 POST + 路径或 body 区分动作」。  
- REST 风格仍值得学：**资源 + HTTP 动词** 语义清晰。

---

## 4. `Optional`、泛型、`record` 在接口里干嘛

- **`Optional`**：可能没有资源（例如按 id 查文章），链式 `map` / `orElseGet` 避免深层 if。  
- **泛型**：`ApiResult<T>` 里 `T` 是 data 的类型，和 TypeScript 泛型类似。  
- **`record`**：不可变 DTO，少写样板代码。

---

## 5. Entity 与 DTO、密码放哪

- **Entity**：内部完整模型，**可含 password**。  
- **DTO（返回前端）**：**不要 password**；登录请求用 **`UserLoginDTO`** 单独接密码。  
- 映射：**MapStruct `UserMapper`** 或手写 `toDto`，行业常见是 MapStruct。

---

## 6. `JwtUtil` 为什么要注入，不能 static

- 密钥、过期时间来自 **配置（`JwtProperties`）**，由 Spring **构造器注入**，便于测试替换。  
- static 工具类适合无状态纯函数；与配置绑定的组件更适合 Bean。

---

## 7. `PostRepository` 的 `@PostConstruct` 谁调用的

- Spring 创建好 Bean 后，**自动调用**一次标注了 `@PostConstruct` 的方法，用于从 classpath 读 JSON 加载内存。重启服务会重新加载。

---

## 8. 文章数据存在哪、重启后为什么没了

- 当前是 **内存 `ConcurrentHashMap`**，**不持久化到文件**。  
- 与前端「页面里的临时变量」类似，**进程结束即丢**。

---

## 9. `ResponseEntity` 和直接返回对象

- 直接返回对象 → 默认 **200**。  
- **`ResponseEntity`**：需要 **201 / 204 / 404** 等时显式指定。  
- 不是「多序列化几次」，而是一层 HTTP 信封 + 一层 `ApiResult`。

---

## 10. 400 / 404 / 401 怎么分工

- **404**：资源不存在（如文章 id 不存在）。  
- **400**：参数错、JSON 解析失败（见 **`ApiExceptionHandler`**）。  
- **401**：鉴权失败（本项目的 Filter 或约定上的 token 问题）。  
- 业务错误也可用 **200 + code ≠ 0**，需团队统一。

---

## 11. React StrictMode 下接口打两次

- 开发环境 **StrictMode** 会双次挂载，导致 `useEffect` 跑两遍。  
- 列表/详情可用 **in-flight Promise 去重**（见前端 `blogApi.ts`）。

---

## 12. CORS 与 Vite 代理

- 开发时可用 **Vite proxy** 把 `/api` 转到 `localhost:8080`，浏览器同源，少踩 CORS。  
- 生产环境需后端 CORS 或同域网关。
