# 计算器练习：如何考虑、设计思路（后端小白向）

## 一、你已有的想法（都是对的）

- 「他是一个类，有加减乘除四个方法」→ 对，这就是计算器的核心能力。
- 「可能有多个变量进行这四个运算」→ 可以，但先不用一次做完。
- 「多个变量又是由 0-9 组成的」→ 那是「输入从哪里来」的问题（键盘、按钮、字符串解析），和「计算逻辑」可以分开想。

## 二、怎么拆就不会懵

### 1. 先抓住「最小单元」

- 计算器最小单元是：**两个数 + 一个运算符 → 一个结果**。  
  例如：`3 + 2`、`10 / 4`。  
  你写的 `add(int a, int b)` 已经就是在做这件事，没问题。

### 2. 「多个变量」怎么处理？

- **简单做法**：每次只算两个数，多步就多次调用。  
  例如算 `1 + 2 + 3`：先 `add(1, 2)` 得到 3，再 `add(3, 3)` 得到 6。  
  不需要一开始就做「一个字符串 1+2+3 怎么解析」，那是后面的进阶。
- **0-9 组成**：说的是用户输入可能是 "123" 这样的数字。在程序里就是 `int` 或 `double` 类型的变量，用你现在的 `add(a, b)` 就行；等以后再做「从字符串解析出数字」也不迟。

### 3. 用「类 + 继承 + 接口」怎么套？

| 概念     | 在计算器里怎么用 |
|----------|------------------|
| **接口** | 约定「计算器能做什么」：有 add、subtract、multiply、divide 四个方法，参数和返回值定死。 |
| **实现类** | 写一个类（如 SimpleCalculator）implements 这个接口，把四个方法的具体实现写出来（你原来的加减乘除逻辑）。 |
| **继承** | 再写一个「科学计算器」类 extends 上面的简单计算器，在保留加减乘除的基础上，多写 sqrt、pow 等方法。 |

这样：**接口** = 约定，**实现类** = 基础计算器，**子类** = 科学计算器，类 + 继承 + 接口就都练到了。

## 三、本目录代码对应关系

- **BasicCalculator.java**：接口，只声明 add/subtract/multiply/divide。
- **SimpleCalculator.java**：实现类，实现四则 + 除零检查。
- **ScientificCalculator.java**：子类，继承 SimpleCalculator，增加 sqrt、pow。
- **CalculatorDemo.java**：用接口类型引用、多态调用，并演示子类用法。

## 四、运行方式

在 `full-stack/java/basics` 目录下：

```bash
javac src/11-practice-calculator/BasicCalculator.java \
      src/11-practice-calculator/SimpleCalculator.java \
      src/11-practice-calculator/ScientificCalculator.java \
      src/11-practice-calculator/CalculatorDemo.java -d out
java -cp out CalculatorDemo
```

## 五、小结

- 先做「两数 + 一运算」，多步就多次调用；不急着做「多个变量、0-9 字符串解析」。
- 接口定义「能做什么」，实现类写「怎么做」，子类在实现类基础上扩展。
- 你原来写的加减乘除就是实现类的核心逻辑，放到 SimpleCalculator 里即可。
