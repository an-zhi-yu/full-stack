# Java 项目规范（本仓库约定）

本仓库内所有 Java 示例与案例统一按以下规范书写，便于学习和后续扩展。

---

## 一、命名

### 1. 仓库名 / 项目根目录名（如 full-stack）

- **推荐：小写 + 连字符**，例如 `full-stack`、`my-blog-backend`。
- 这样和常见 Git 仓库、URL、命令行路径一致，避免大小写带来的跨平台问题。

### 2. Java 项目/模块名（如 HelloWorld）

- **大驼峰（PascalCase）**，例如 `HelloWorld`、`BlogApi`。
- 一个“项目”通常对应一个可运行或可被引用的功能单元（一个文件夹，里面可能有多个类）。

### 3. 类名（Class）

- **大驼峰（PascalCase）**，例如 `Main`、`UserService`、`OrderController`。
- 文件名必须与 **public 类名完全一致**：`public class Main` → 文件名为 `Main.java`。

### 4. 方法名、变量名

- **小驼峰（camelCase）**，例如 `getName()`、`userId`、`orderList`。
- 常量（static final）：**全大写下划线**，例如 `MAX_SIZE`、`DEFAULT_TIMEOUT`。

### 5. 包名（package，若使用）

- **全小写**，多词用点号分隔，例如 `com.example.blog`、`cn.demo.utils`。
- 常见习惯：反写域名 + 项目/模块，如 `com.anzhiyu.fullstack.hello`。

## 二、访问修饰符与 static

### 访问修饰符（谁可以访问）

| 修饰符 | 谁可以访问 |
|--------|------------|
| **public** | 任意类（同包、其他包都可） |
| **protected** | 本类 + 同包类 + 子类（含其他包里的子类） |
| **不写（默认）** | 本类 + 同包类（包私有） |
| **private** | 仅本类内部 |

- 字段常用 **private** 封装，通过 public 的 getter/setter 暴露。
- 构造方法一般 **public**，否则外部无法 new。

### static 与 public 的区别

- **static**：和「要不要对象」有关。有 static → 方法**属于类**，用 **类名.方法名(参数)** 调用，**不需 new**。没有 static → 方法**属于对象**，要先 new 再 **对象.方法名(参数)**。
- **public / private**：和「谁可以访问」有关。可组合：`public static` = 任意处可调且用类名调；`private static` = 仅本类可调、用类名调。
- 更通俗的解释（含前端类比、四种写法对照）：见 **docs/static与包-通俗解释.md**。

### 构造方法重载

- 多个构造方法同名（与类名相同）、参数列表不同，叫「重载」。`new Book()` 调无参构造，`new Book("a","b",1.0)` 调三参构造。

## 三、目录结构

- 源码放在 **`src/`** 下，例如 `HelloWorld/src/Main.java`。
- 编译输出放在 **`out/`** 或 **`target/`**，不提交到 Git（由 .gitignore 忽略）。
- 单项目示例：
  ```
  HelloWorld/
    src/
      Main.java
      Run.java
    out/          # 编译产物，不提交
  ```
- 若使用 Maven/Gradle，则按标准布局（如 `src/main/java`）即可。

---

## 四、入口与运行方式

- 可执行程序的入口固定为：`public static void main(String[] args)`。
- 一个项目里可以有多个带 `main` 的类（如 `Main`、`Run`），运行哪个就执行：`java -cp out 类名 [参数...]`。
- 在别的类里调用入口：`Main.main(args);` 或 `Main.main(new String[]{"a","b"});`（参数为字符串数组）。

---

## 五、注释与文档

- 类、公开方法上方用 **Javadoc**（`/** ... */`）简要说明用途、参数、用法。
- 复杂逻辑或“为什么这样写”可在行内用 `//` 或块注释 `/* ... */` 说明。
- 本仓库内示例会尽量带中文注释，方便新手对照理解。

---

## 六、格式与风格

- 缩进：4 个空格（或与现有文件保持一致）。
- 大括号：与当前项目一致即可，如 `public void foo() {` 换行写方法体。
- 行宽：建议单行不超过 120 字符，过长时换行缩进。

## 八、和前端（JS/TS）的常见差异

### 传参：命令行 vs 方法里

- **不一定都在方法里调用**。命令行传参很常见（脚本、工具、服务启动参数）。在方法里 `Main.main(new String[]{...})` 适合写死参数、测试、或由其他代码构造参数。
- 两种方式按需选即可。

### 数组：为什么是 `new String[] { "你好", "世界" }` 而不是 `["你好", "世界"]`？

- **前端**：数组有字面量，直接写 `["a", "b"]` 或 `[1, 2, 3]`。
- **Java**：没有数组字面量，数组必须「创建」出来。写法是：
  - `new 元素类型[] { 元素1, 元素2 }`  
  例如 `new String[] { "你好", "世界" }` 表示「创建一个 String 数组对象，里面有两个元素」。
- 所以和前端对比：前端用 `[xx, xx]` 一步到位；Java 用 `new String[] { "你好", "世界" }` 一步到位，只是语法不同。

### 「new String[] 是数组，为什么后面像对象？」

- 在 Java 里，**数组本身就是一种对象**（引用类型）。`new String[] { ... }` 就是在「 new 一个数组对象」，`{ "你好", "世界" }` 是给这个数组赋初值，不是「对象字面量」。
- 可以类比：前端里 `[]` 也是对象（typeof [] === 'object'）；Java 里数组也是对象，只是创建方式用 `new 类型[] { ... }`，没有 `[]` 这种简写。
