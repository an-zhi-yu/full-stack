# 项目目录结构与技术栈说明

> 面向「全栈学习 + 可上线项目」的统一目录设计，便于扩展后端模块、多语言学习与前后端打包部署。

---

## 一、推荐目录结构

以 **full-stack** 为仓库根，前端、后端、各语言学习、部署脚本分层清晰，便于后续加模块。

```
full-stack/
├── frontend/                          # 前端（所有前端项目集中在此）
│   ├── blog/                          # 博客前台（React + Ant Design）
│   │   ├── src/
│   │   ├── public/
│   │   ├── package.json
│   │   ├── vite.config.ts
│   │   └── 前端代码规范.md
│   ├── admin/                         # 示例：后台管理
│   │   └── ...
│   └── 说明.md                        # 前端技术栈与启动说明
│
├── backend/                           # 后端服务（按项目分子目录）
│   ├── blog-api/                      # 示例：博客 API（如 Spring Boot）
│   └── 说明.md
│
├── java/                              # Java 学习与示例
│   ├── basics/
│   ├── 环境与搭建.md
│   ├── 编码规范.md
│   ├── 常用API速查.md
│   └── ...
│
├── python/                            # Python 学习与实践
├── linux/                             # Linux 学习与脚本
├── deploy/                            # 打包与上线相关
├── docs/                              # 仓库级文档
├── .gitignore
├── 说明.md                            # 仓库总览
├── 仓库目录说明.md                    # 目录划分说明
└── 项目结构与技术栈.md                # 本文件
```

### 前端代码写在哪？

- **所有前端项目** 放在 **`frontend/`** 下，按项目建子目录，例如：
  - `frontend/blog/`：博客前台
  - `frontend/admin/`：后台管理
- 每个项目是**独立工程**（自己的 `package.json`、构建命令），便于单独打包、单独部署。

---

## 二、扩展性说明

| 类型 | 放置位置 | 说明 |
|------|----------|------|
| 前端新项目 | `frontend/<项目名>/` | 新起一个目录，独立 package.json，互不影响 |
| 后端新服务 | `backend/<服务名>/` | 如 blog-api、admin-api，可 Java/Node/Python 混用 |
| Java 学习 | `java/<主题>/` | basics、collections、io、concurrency 等；文档放 java/ 下 |
| Python 学习 | `python/<主题>/` | basics、scripts、flask-fastapi 等 |
| Linux 学习 | `linux/scripts/`、`linux/notes/` | 脚本与笔记 |
| 部署与运维 | `deploy/` | Docker、Nginx、构建脚本，与业务代码分离 |

## 三、前后端打包与上线支持

### 3.1 构建产物位置（便于上线）

- **前端**：每个项目在各自目录下执行 `pnpm build`（或 `npm run build`），产物一般在 `frontend/<项目名>/dist`（Vite）。
- **后端**：如 Java 用 Maven/Gradle，产物在 `backend/<服务名>/target/*.jar` 等。

建议在 **`.gitignore`** 里忽略：`frontend/*/dist`、`frontend/*/node_modules`、`backend/*/target` 等。

### 3.2 部署方式（都支持）

1. **静态 + 接口分离**：前端 dist 用 Nginx/对象存储；后端 jar 或 node 用 systemd/docker 跑，Nginx 反代。
2. **Docker**：`deploy/docker` 放 Dockerfile、docker-compose。
3. **CI/CD**：GitHub Actions / GitLab CI 构建 frontend/blog、backend/blog-api 后部署。

## 四、前端技术栈方案与依赖整理

博客前端 **frontend/blog** 已采用：**React 18 + Vite + React Router + Zustand + Ant Design + Axios + TypeScript + ESLint + Prettier + Husky + Vitest**。  
更多方案（Vue、Nuxt、纯静态等）与依赖清单见上文各方案表；规范细节见 **frontend/blog/前端代码规范.md**。

---

## 五、小结

1. **前端代码位置**：统一放在 **`frontend/<项目名>/`**，一个项目一个子目录，独立构建。
2. **扩展**：后端在 `backend/`，Java/Python/Linux 学习在对应目录，Java 学习类文档放在 **java/** 下。
3. **打包上线**：各子项目独立可构建；Nginx、Docker、CI/CD 均支持。
4. **文档**：根目录用中文命名（说明.md、仓库目录说明.md、项目结构与技术栈.md），Java 相关见 **java/** 下中文文档。
